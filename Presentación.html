Paso 01

Creación
Bucket

Se configuró el bucket bucket-bigdata-diego en Google Cloud Storage, con ubicación multi-regional en Estados Unidos y clase Standard. Se cargaron los 12 archivos Parquet de viajes mensuales 2025 (~800 MB total) y el archivo CSV de zonas.

GCSStandard ParquetCSV~800 MB

Paso 02

Creación
Tablas

Se crearon tablas externas en BigQuery: trips_external (Parquet) y locations_external (CSV). Se construyó la vista trips_hourly agrupando viajes por hora con TIMESTAMP_TRUNC para alimentar el modelo predictivo.

BigQueryExternal Tables trips_hourlyTIMESTAMP_TRUNC
Tablas externas BigQuery
Paso 03

Outliers

Se detectaron valores extremos: tarifa anómala de $863,372 y distancias >100 millas. Usando APPROX_QUANTILES se calcularon percentiles p50, p90, p95 y p99 para fijar rangos válidos: distancia 0.1–79.3 mi y tarifa $2.50–$833.

APPROX_QUANTILESp50·p90·p95·p99 trip_distancefare_amount
Detección de outliers con percentiles
Paso 04

Limpieza
de Datos

Se generó la tabla main_trips filtrando rangos válidos, 1–6 pasajeros, duración 1–180 min y tipos de pago 0–6. Se enriqueció con zonas pickup/dropoff mediante INNER JOIN y se calcularon fare_per_mile y tip_pct.

Paso 05

Creación
Modelo

Se entrenó un modelo ARIMA_PLUS en BigQuery ML (forecast_trips). La columna temporal es pickup_hour y el objetivo es total_trips, a partir de la vista trips_hourly. Permite predecir la demanda horaria futura de taxis.

BigQuery MLARIMA_PLUS forecast_tripsSeries de tiempo
Modelo ARIMA_PLUS · BigQuery ML
Paso 06

Consulta
Programada

Se configuró la scheduled query "prediccion de viajes de TAXI" que se ejecuta automáticamente cada hora en BigQuery. Garantiza que el modelo ARIMA_PLUS actualice sus predicciones de forma continua, manteniendo el dashboard con datos frescos.
